# 보안 정책 (Security Policy)

이 저장소는 삼성증권 영크리에이터 15기 4조의 **OpenCode 연동 MCP 개발** 프로젝트입니다.
MCP tool은 **로컬 파일 시스템에 접근하는 코드를 다루는 것을 전제로 설계**합니다.
보안 이슈는 일반 버그보다 높은 우선순위로 처리합니다.

## 취약점 제보 방법

보안 취약점을 발견하셨다면 **공개 이슈로 올리지 마시고** 아래 채널로 비공개 제보해 주세요.

- 이메일: **team4youngcreator@gmail.com**
- 제목에 `[SECURITY]`를 붙이고, 다음을 포함해 주세요.
  - 취약점 설명과 영향 범위
  - 재현 절차 (가능하면 단계별로)
  - 발견한 브랜치/커밋

제보를 받으면 **3일 이내에 1차 회신**하는 것을 목표로 합니다. 수정이 완료되어 배포되기
전까지는 취약점 내용을 공개하지 말아 주세요.

## 지원 범위

| 브랜치 | 지원 여부 |
| --- | --- |
| `main` (배포 버전) | ✅ |
| `develop` (개발 통합) | ✅ |
| 작업 브랜치 (`feature/*`, `fix/*` 등) | ❌ — develop 머지 후 처리 |

## 비밀 정보 관리 원칙

- **API 키·비밀번호·서비스 키는 절대 커밋하지 않습니다.** 모든 비밀 값은 `.env`로만
  관리하고, 플레이스홀더만 담긴 `.env.example`을 활용합니다.
- MCP 서버는 **외부 상용 LLM API·Web/외부 API를 호출하지 않습니다** (과제 제약).
  LLM 추론은 로컬 Ollama 모델로만 수행합니다.
- MCP tool이 로컬 파일에 접근할 때는 허용된 디렉토리 밖(경로 탈출, `..` 등)으로
  나가지 않는지 확인합니다.
- 에러 메시지·로그에 개인 정보·비밀 값을 출력하지 않습니다.

## 운영 중인 보안 장치

- **브랜치 보호**: GitFlow(`feature → develop → main`). `main` 직접 커밋 금지,
  모든 변경은 PR + 최소 1명 리뷰 승인 후 머지.
- **GitHub 자동 스캔**: Secret scanning(비밀 푸시 감지) + Push protection,
  Dependabot alerts(의존성 취약점) 활성화.

자세한 협업 규칙은 [`CONTRIBUTING.md`](CONTRIBUTING.md)를 참고하세요.
